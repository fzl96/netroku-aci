# Fumadocs `/docs` Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a public documentation site at `/docs` inside the existing Next.js app using Fumadocs MDX, with stub content for user guide and admin sections.

**Architecture:** Fumadocs MDX compiles `.mdx` files from `content/docs/` at build via a `createMDX()` wrapper in `next.config.ts`. A generated `.source/` index is loaded by `src/lib/source.ts` and consumed by a `DocsLayout` at `src/app/docs/layout.tsx` + a catch-all page at `src/app/docs/[[...slug]]/page.tsx`. The route lives outside `(app)` so it's public (no auth).

**Tech Stack:** fumadocs-ui, fumadocs-core, fumadocs-mdx, @types/mdx, Next.js 16, Tailwind v4

## Global Constraints

- Next.js 16 with `reactCompiler: true` — must be preserved in config.
- Tailwind v4 with `@import "tailwindcss"` syntax — no v3 `tailwind.config.js`.
- Package manager: `pnpm` (lockfile is `bun.lock` but scripts use `bun`; install with `pnpm add`).
- Path alias: `@/*` maps to `./src/*`.
- The `.source/` directory generated by fumadocs-mdx must be added to `.gitignore`.
- The `content/` directory must NOT be gitignored (unlike `docs/`).

  baseUrl: "/docs",

- [ ] **Step 2: Create `src/app/docs/layout.tsx`**

This layout wraps all `/docs` pages with the Fumadocs sidebar. It uses `RootProvider` with `theme.enabled: false` because the root layout already provides next-themes `ThemeProvider`.

```tsx
import { RootProvider } from "fumadocs-ui/provider";
import { DocsLayout } from "fumadocs-ui/layouts/docs";
import { source } from "@/lib/source";
import type { ReactNode } from "react";

export default function Layout({ children }: { children: ReactNode }) {
  return (
    <RootProvider theme={{ enabled: false }}>
      <DocsLayout tree={source.getPageTree()}>
        {children}
      </DocsLayout>
    </RootProvider>
  );
}
```

- [ ] **Step 3: Create `src/app/docs/[[...slug]]/page.tsx`**

```tsx
import {
  DocsPage,
  DocsBody,
  DocsTitle,
  DocsDescription,
} from "fumadocs-ui/layouts/docs/page";
import { source } from "@/lib/source";
import { notFound } from "next/navigation";
import defaultMdxComponents from "fumadocs-ui/mdx";

export default async function Page(props: {
  params: Promise<{ slug?: string[] }>;
}) {
  const params = await props.params;
  const page = source.getPage(params.slug);
  if (!page) notFound();

  const MDX = page.data.body;

  return (
    <DocsPage toc={page.data.toc}>
      <DocsTitle>{page.data.title}</DocsTitle>
      <DocsDescription>{page.data.description}</DocsDescription>
      <DocsBody>
        <MDX components={{ ...defaultMdxComponents }} />
      </DocsBody>
    </DocsPage>
  );
}

export function generateStaticParams() {
  return source.getPages().map((page) => ({
    slug: page.slugs,
  }));
}

export async function generateMetadata(props: {
  params: Promise<{ slug?: string[] }>;
}) {
  const params = await props.params;
  const page = source.getPage(params.slug);
  if (!page) notFound();

  return {
    title: page.data.title,
    description: page.data.description,
  };
}
```
