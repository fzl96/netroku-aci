The dashboard is the landing page after sign-in. It aggregates data from all configured APIC hosts into a single operations view — no per-page navigation needed to get an operations snapshot.

## Global posture

The top-left banner summarises the overall fabric state:

| Label | Meaning |
|---|---|
| **Stable** | No active risk signals |
| **Degraded** | Interfaces with error counters |
| **Needs attention** | Failed hardware or offline nodes |

## Headline stats

Four clickable stat tiles sit to the right of the posture banner. Each links to the relevant detail page:

| Stat | Detail shown | Links to |
|---|---|---|
| Nodes online | Online/total ratio | Nodes |
| Failed hardware | Failed PSU / fan breakdown | Nodes (components view) |
| Active endpoints | Total learned count | Endpoints |
| Interfaces with errors | Operationally down count | Interface Health |

Tiles are colour-coded: green for healthy, amber for warning, red for critical, grey for no data.

## Metric cards

Below the headline strip, three cards give a more detailed breakdown:

- **Endpoints** — active count, historical count, unique VLAN count, node/interface spread
- **Interfaces** — total tracked, interfaces with recent errors, oper-down and admin-down counts
- **Nodes & Hardware** — online/total nodes, failed component count, leaf/spine/controller breakdown

## Attention required

Lists only the items that need action, ordered by severity:

1. Failed hardware (PSU or fan)
2. Offline nodes
3. Interfaces with error counters
4. Operationally down interfaces

If nothing needs attention, a green "No active attention items" banner is shown instead.

## APIC host coverage

A table on the right shows one row per configured APIC host with:
- Active endpoint count
- Nodes online ratio
- Failed hardware count
- Age of the freshest data from that host

The freshness chips above the table show how long ago each dataset (Endpoints, Interfaces, Nodes) was last synced across all hosts.

## Data freshness

The header bar shows two chips: the number of configured APIC hosts and the age of the newest data point across all datasets. If this reads "Never synced", no resync has been run yet — see [Scheduled Resync](/docs/admin/scheduled-resync).
