---
title: Health Scores
description: ACI fabric health score monitoring.
---

The health scores page shows the ACI health score for each fabric object — the overall fabric, individual nodes, and tenants. Scores are pulled from APIC and stored as a snapshot on each resync.

## Score bands

ACI health scores run from 0 to 100. Netroku ACI uses three bands:

| Band | Range | Colour |
|---|---|---|
| **Good** | 90–100 | Green |
| **Fair** | 70–89 | Amber |
| **Poor** | 0–69 | Red |

A score of `—` means no data has been synced for that object yet.

## Columns

| Column | Description |
|---|---|
| Scope | Object type: `fabric`, `node`, or `tenant` |
| Name | Object name (node ID or tenant name; blank for fabric scope) |
| Node | Node ID for node-scope objects |
| Score | Current health score (0–100) |
| Max severity | Highest-severity active fault affecting this object |
| Last seen | When this object was last present in a resync |

## Scopes

- **Fabric** — single aggregate score for the entire ACI fabric
- **Node** — individual health score per leaf, spine, or controller node
- **Tenant** — health score per tenant

The dashboard shows the **worst node/tenant score** as its primary health indicator. An object is considered degraded when its score drops below 90.

## Resync

Click **Resync** to pull a fresh health score snapshot from APIC. You will be prompted for APIC credentials. The resync updates fabric, node, and tenant scores in one operation.
